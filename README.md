# DevOps СИТУ Лабораторная работа №5: Создание Docker-образа с Nginx и запуск контейнера

## Описание

Цель лабораторной работы - создать собственный **Docker-образ** с веб-сервером **Nginx**, который отдает статическую HTML-страницу с приветствием, запустить контейнер и обеспечить доступ к нему через проброшенный порт **54321**. Запуск контейнера автоматизирован с помощью shell-скрипта.

## Структура проекта

```text
.
├── deploy.sh
└── nginx
    ├── dockerfile
    ├── index.html
    └── nginx-conf
        └── nginx.conf
```

## Содержимое файлов

### `deploy.sh`

Shell-скрипт выполняет следующие действия:

1. Останавливает и удаляет существующий контейнер (если он существует).
2. Собирает Docker-образ.
3. Запускает контейнер с пробросом порта **54321 → 80**.
4. Выводит список контейнеров.
5. Проверяет доступность веб-сервера с помощью `curl`.
6. Отображает последние записи журнала контейнера.

### `dockerfile`

Dockerfile выполняет следующие действия:

1. Использует официальный образ **Nginx** в качестве базового.
2. Копирует HTML-страницу в каталог `/var/www/html`.
3. Копирует пользовательскую конфигурацию Nginx.
4. Проверяет корректность конфигурации Nginx.
5. Запускает веб-сервер в режиме foreground.

### `index.html`

Статическая HTML-страница, отображающая приветственное сообщение.

```html
<h1>Hello from Docker + Nginx!</h1>
```

### `nginx.conf`

Конфигурация Nginx:

- прослушивает порт **80**;
- использует каталог `/var/www/html` в качестве корневого;
- отдает статические файлы.

## Запуск

Сделать скрипт исполняемым:

```bash
chmod +x deploy.sh
```

Запустить развертывание:

```bash
./deploy.sh
```

## Результат

После успешного выполнения скрипта:

- создан Docker-образ с веб-сервером **Nginx**;
- запущен контейнер `nginx-cont`;
- порт **80** контейнера проброшен на порт **54321** хоста;
- при обращении к `http://<IP_хоста>:54321` отображается приветственная HTML-страница;
- в консоль выводятся результаты проверки работы контейнера и журналы Nginx.
